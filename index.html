Some Question and Answer

JavaScript is an interpreted programming language. so, It doesn't need to be compiled into binary code for execution. When a JavaScript file has to be loaded, JavaScript engine break down the code line by line (top to bottom) and convert into machine code. But it doesn't run immediately. HTML and CSS DOM create the webpage first. Then JavaScript engine run the JavaScript file.

JavaScript can do just one thing at a time. so, it creates a Task Queue for execute all the tasks. at first, tasks get pushed into the queue, Web API's execute tasks , then go back to the call stack. When the stack is clear, Event loop check queue for available callbacks. It pushed callbacks to the call stack and waits for execution.

Local storage has much greater storage capacity. It reduce traffic between client and server as the data is not sent back to the server on every HTTP request. Data remains on the local storage until user delete it. Saved data available for all current and future visit to the site.

Session storage is only available for current window. once the window is closed, the data from session storage has been deleted.
